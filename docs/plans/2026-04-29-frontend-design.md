# Frontend Design

## Goal

在不改变现有后端业务边界的前提下，增加一个最小前端，方便直接观察：

- 创建规划
- interrupt 补充信息
- 完成后查看攻略
- 基于反馈做局部重规划

## Options

### Option 1

静态单页 `HTML + CSS + JS`

- 优点：最小依赖、最容易学、最贴近当前项目规模
- 缺点：没有组件体系，不适合继续长大

### Option 2

FastAPI 模板渲染

- 优点：仍然轻量
- 缺点：这里并不需要服务端模板，反而多一层抽象

### Option 3

独立前端工程

- 优点：更现代
- 缺点：会把学习重点从 Agent 工作流转移到前端工程化

## Decision

选择 Option 1。

## UI Scope

页面只保留 3 个交互区：

1. 旅游需求输入
2. 缺失信息补充
3. 攻略反馈

结果区只负责显示：

- session id
- status
- 当前问题或攻略

## Visual Direction

采用“旅行档案 / 任务简报”风格：

- 暖纸色背景
- serif 标题
- 工具化正文
- 左输入、右结果

这样既有辨识度，也不会喧宾夺主。
